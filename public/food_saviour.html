Introduction

Food Saviour is a 3 person prototyping project. It is for on of our academic module that focuses on creating user interfaces that fits the user requirements and is easy to navigate and use. As we all know, if we create complex interfaces that users cannot navigate or contains broken navigation, the user simply cannot use it. Therefore, Food Saviour is a project that aims to provide users with an easy to use interface as well as serves the purpose of the application that fits the user needs.

The prototype for Food Savior is created using marvelapp. Food Savior initially started as a mobile app prototype as we will be required to deal with the small screen that mobile devices provides to fully utilize the space in it. The final prototype changes the application to be tablet based to cater for more features, functions and as well as efficient usage of the spaces that tablet provides.

Features

Login

Just like all applications, Food Savior also has a Login page as a landing page right after the splash screen (Refer to image above). The user input their email and password to enter the application.

For our prototype we only focus on food donater and food deliveryman. The interface for admins and food donation receiver will operate in a totally separate application which is not yet developed.

The register button also lies in the Login Page. It will help facilitate food donation donor and deliveryman to register an account.

Delivery Page

One of the highlighted features was the food delivery feature. It allows the user to choose which were the food delivery service required nearby. Volunteers can then help to deliver the food to the receivers doorstep.

The delivery page shows the location to deliver to as well as a list of food to be deliver.

To-Buy List

We also included a feature for the users who wish to donate food to be able to directly use the app for their to-buy list. For the sake of convenience, the user can also input the food expiry date for them to keep a record of the expiry date for the food they buy.

Chats Page

In order to handle the event of the deliveryman was unable to locate the unit to deliver to or any unwanted circumstances, the chats feature came in to overcome the issue. It allows the users to chat through it for important informations to pass through or any last minute cancellations.

Account Page

Last but not least the account page which allows the user to edit their account details and choose their prefered colour theme. This page also display the rating of the user by the food recipients and donation drives.

That basically wraps up all the main features of the prototype. Want to know more? Check this out.
