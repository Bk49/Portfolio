Introduction


Shopmania was a pair programming work between me and one of my best teammate Khye Lerk's work In this project, we were using MVC framework to create a dynamic web project which is a website that if used for e-commerce mainly selling computer accessories. It is created using JSP pages, JavaBeans and MySQL database. The database is hosted on Heroku using the ClearDB to keep it running at all times.

Shopmania is also hosted on Azure. However due to some part of the project was not uploaded by my teammate, the website will contain a few broken links.

Features

Login and Home

Start off with the home page which serves as a landing page for all the users. Public users need not to register or login to an account to start browsing for products.

It contains all the product categories that the user can browse. Below of it displays 10 random products selected from all the products available in the store.

From the home page, there will be a Login button at the top right corner where user can proceed to input their login details to login as a member or admin. The user will be redirected back to home page with the user session if the user is just a normal member. Admins will instead be redirected to an alternative admin page.

For new users, they can also register for a new account at the login page with their details.

Admin Pages

One of the administrator power is their ability to add new products, discount codes or users. Image at the left shows how the adminstrator can a new product using the website interface to input new rows into the database.

Admins can also use the website to modify and edit products that were previously in the database already and ammend them.

Since every product come with their unique ID, their names can be changed to be the same as others as well and also treated as non-duplicated and separated objects. This can especially help on products with same names but comes in different conditions such as colors etc. which are treated as individual products.

The last important administrator power is their ability to delete products, discount codes and users in the database completely.

Since deleted items can no longer be retrieved again, to prevent accidental lost of data, a confirmation will be done to confirm with the admin whether to remain the items or not. It will display the target to be deleted to ensure it was the item to be deleted to prevent deleting of items that has similar or identical names.

This image shows an overall look of the admin view. An important point to mention is regarding the security of the admin pages since it is in the same domain that can be accessed by the users.

Since prepared statements are used for the whole project, SQL injection is not possible. The admin pages are also safe from Broken Access Control since for all the admin pages, the user session will be checked for the role first before being displayed. If the user was a member, they will never make it to access these pages.

Shopping

For all users, they can navigate throughout the website to check out all the products avaialble in Shopmania.

Items ranges from computer graphics card to gaming mouse pads.

Users with accounts will gain the ability to add items to their shopping cart and continue shop for other items. It will show how many products of such type is already in the cart if the users already added at least one of them into the cart.

Each item before added into the cart will display its product details and price.

After user has done shopping for products they've wanted they can move on to checkout their shopping cart. Shopmania will display all the products they that is going to be checked out. The user can choose to remove the products here before continue with check out.

This is also where the user can use the discount code to reduce the price to be paid.

After the user check out, the purchase history will be saved. The user can now proceed to make payment based on payment details they have previously made or create payment using new payment details. By ticking the checkbox, they can park they payment details in the database to be used during their next purchase.

Since the shopping cart is session based, the user can choose to save the products to be bought in the purchase history to complete their payment afterwards when they want to.

It is also good to mentioned all payment details will be validated to be in the correct length or format before proceeding to make the payment and check out with the items.

Profile

Last but not least the user can change their profile details by clicking at their profile icon.

From here, they can edit their name, email, profile picture and also their phone number.
